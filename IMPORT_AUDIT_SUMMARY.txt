================================================================================
                     WHY THIS HAPPENED
================================================================================

Python Import Semantics:
  from .alarms import func1, func2  # Imports ONLY func1 and func2
  from . import alarms              # Imports the alarms MODULE object

The persistence and models __init__.py files use the first pattern,
making functions available but not the module objects themselves.

To support BOTH usage patterns:
  from persistence import get_alarm_rules  # Direct function access
  from persistence import alarms           # Module access
  
You need BOTH import styles in __init__.py:
  from .alarms import get_alarm_rules, ...  # For direct access
  from . import alarms                       # For module access
